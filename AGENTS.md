# AGENTS

本檔案定義此專案中的固定工作規則。

## 語言規則

- 預設使用繁體中文。
- 優先使用台灣常用詞彙與表達方式。
- 除非使用者明確要求，否則不要改用簡體中文或中國用語。

## 指令紀錄規則

- 每次收到與本專案有關的新使用者指令，都必須先追加記錄到 [INSTRUCTIONS.md](/c:/Users/linyuxuan/Desktop/travel-claude+gpt/INSTRUCTIONS.md)。
- 紀錄內容必須包含：日期、時區、使用者指令、任務類型、建議執行者、判斷原因、執行備註、最後決策、狀態、輸出或異動檔案、下一步。
- 新紀錄必須追加在 `INSTRUCTIONS.md` 最下方，不可覆蓋既有紀錄。
- 若任務執行方向在過程中改變，必須同步更新 `INSTRUCTIONS.md` 的 `最後決策` 與 `執行備註`。

## 執行者決策規則

- 由使用者自行決定任務要交給誰處理。
- 若使用者未明確指定，紀錄中的 `建議執行者` 只作為當次執行紀錄，不視為固定分流規則。
- 不要在專案文件中建立預設的 Claude / Codex 任務分流規則，除非使用者另外要求。

## 執行原則

- 先記錄，再執行。
- 若沒有檔案異動，也要在紀錄中明確註記 `無檔案異動`。
- 若工作卡住，必須在紀錄中寫明阻塞原因。

## Claude 專屬規則

- 每次收到使用者指令時，必須在 `INSTRUCTIONS.md` 最下方追加一筆紀錄，依照該文件既有風格與模板格式撰寫。
- 紀錄中的 `建議執行者` 須依當次實際執行者填寫，且不得寫成 `Claude + Codex`。
- 不要改動 `INSTRUCTIONS.md` 的既有內容，僅從最下方追加。
- 全程使用繁體中文與台灣用語。

## Codex 專屬規則

- 本專案以 `AGENTS.md` 作為 `agent.md` 的規則文件使用。
- 每次收到使用者對 Codex 下達的專案指令時，必須先在 `INSTRUCTIONS.md` 最下方追加一筆新紀錄，再開始執行。
- 不可重寫整份 `INSTRUCTIONS.md`，只能從文件最下方開始寫入，並沿用整份文件既有風格。
- 每筆由 Codex 追加的紀錄，都要明確註明本段指令與決策是由 `CODEX` 進行。
- 全程使用繁體中文與台灣用語。
